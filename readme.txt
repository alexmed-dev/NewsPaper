По итогу работы в модуле D6 вы должны были усовершенствовать ваше новостное приложение:

Все данные с реальными электронными адресами, паролями и т.д. хранятся в файле .env в корне проекта и не загружаются в удаленный репозиторий.

1. В категории должна быть возможность пользователей подписываться на рассылку новых статей в этой категории.
   На странице новости (например: http://127.0.0.1:8000/news/35) вверху
	выведен список ссылок на категории, к которым эта новость относится.
   При переходе по ссылке на категорию выходит страница новостей в этой категории
	(например, http://127.0.0.1:8000/news/category/1/),
	а вверху ссылка "Подписаться".
   Нажав на ссылку пользователь подписывается на данную категорию, после чего:
	* ему на почту приходит соответствующее уведомление;
	* на странице данной категории вместо ссылки "Подписаться" теперь выходит ссылк "Отписаться";
	* на странице профиля (http://127.0.0.1:8000/news/profile) выходит список категорий,
	  на которые пользователь подписан с сылкой "Отписаться" для каждой.

2. Если пользователь подписан на какую-либо категорию, то, как только в неё добавляется новая статья, 
   её краткое содержание приходит пользователю на электронную почту, которую он указал при регистрации. 
   В письме есть гиперссылка на саму статью, чтобы он мог по клику перейти и прочитать её.

3. Если пользователь подписан на какую-либо категорию, то каждую неделю ему приходит на почту список новых статей, 
   появившийся за неделю с гиперссылкой на них, чтобы пользователь мог перейти и прочесть любую из статей.

4. При регистрации в приложении на почту пользователю приходит приветственное письмо с ссылкой на активацию аккаунта.
   После перехода по ссылке и подтверждения пользователь может зайти на сайт новостей.
